Bot Flow — Overview

Every session cycles through three states. The bot never skips ahead — it finishes each state before moving to the next.

WATCHread crash history
→
TRIGGERcondition met
→
BETup to N rounds
→
WINdeficit drops
or
EXHAUSTall N rounds lost
→
WATCHagain
  1. Watch — the bot reads every crash result. No money is placed. It is looking for a pattern that matches your trigger condition.
  2. Trigger — when the condition fires (e.g. last crash was above 9×, or 8 consecutive crashes all stayed below 3×) the bot enters bet mode for the next Max Bet Rounds rounds.
  3. Bet — both Panel 1 and Panel 2 are bet every round. Each panel has its own auto-cashout set once at startup. The bet amounts change each round based on the recovery mode you configured.
  4. Result — after each round the bot updates the deficit for each panel, then either continues betting (if rounds remain) or returns to watch mode.
  5. Cooldown (optional) — after every burst, the bot can skip N watch rounds before looking for a new trigger. Prevents re-entering immediately after a loss run.
Two panels, always active

Panel 1 targets a high cashout (default 6×) and carries the main recovery. Panel 2 targets a lower cashout (default 3×) and acts as a hedge — it wins more often but pays less. Each panel tracks its own deficit independently.

Recovery — The Core Idea

When a round is lost, the bot records the loss as a deficit. The next bet is sized to win back that deficit plus a small profit margin. This is called martingale-style recovery.

The Recovery Scope controls which losses each panel tries to recover. The Recovery Mode (Individual / Combined / Percentage) controls how much it targets each round.

Bet sizing formula

bet = (target + recovery_profit_margin) ÷ panel_cashout

Example: target = 500 KES, margin = 5, P1 cashout = 6×
bet = (500 + 5) ÷ 6 = 84.17 KES

If that round wins at 6×: net gain = 84.17 × (6−1) = 420.83 KES ≈ 500 KES recovered.

Recovery Scopes

Each panel (P1 and P2) has its own scope setting. They work independently unless an assist rule is enabled.

Scope 1
Individual (default)
Each panel is responsible for its own losses only. P1 sizes its bet to recover P1 losses. P2 sizes its bet to recover P2 losses. Assist rules can let a clean panel target part of the other panel's deficit.
P1 target = P1_deficit, or P2_deficit × assist_pct when P1 is clean
P2 target = P2_deficit
On P1 assist win → P2 deficit drops by P1 net gain.
On P2 win → P2 deficit clears. P1 deficit unchanged.
Scope 2
Combined
One panel takes responsibility for recovering both panels' losses together. Useful when one panel consistently wins and you want it to cover the other's losses too.
// P1 set to Combined:
P1 target = P1_deficit + P2_deficit
P2 target = P2_deficit (unchanged)
On P1 win → BOTH deficits clear to 0.
On P2 win → P2 deficit clears. P1 deficit unchanged.
Scope 3
Percentage
Instead of recovering everything at once, the panel tries to recover a configured percentage of the combined deficit each round. The last round in a burst always goes for 100% of whatever remains — one final all-in attempt.
// Rounds 1 to N−1:
target = (P1_deficit + P2_deficit) × (pct ÷ 100)

// Round N (last):
target = P1_deficit + P2_deficit ← 100%
On win: deficit reduces by target amount (not zeroed).
On loss: deficit grows by the bet placed. Next round recalculates from new total.

Percentage Mode — Step-by-Step Trace

This is the most powerful but most nuanced mode. Walk through both the all-wins and loss-then-win scenarios below.

Setup

Starting deficit: 1,000 KES  ·  Recovery %: 50%  ·  Max bet rounds (N): 3  ·  P1 cashout: 6×  ·  Profit margin: 5 KES

Scenario B — Round 1 lost, rounds 2 and 3 win

Trace · same setup — round 1 loses
Round Deficit before bet Target Bet placed Outcome Deficit after
1 of 3 1,000 KES 50% → 500 (500+5)÷6 = 84.17 LOSS 1,000 + 84.17 = 1,084.17 KES

Key behaviour

When a round is lost, the actual bet amount (not the target) is added to the deficit. This means subsequent rounds must work harder. The 50% target is always recalculated from the new, higher total deficit — not the original.

Scenario C — All three rounds lost (burst exhausted)

Trace · all rounds lose — bot returns to watch mode
Round Deficit before Bet placed Outcome Deficit after
1 of 3 1,000 KES 84.17 LOSS 1,084.17 KES
2 of 3 1,084.17 KES 91.18 LOSS 1,175.35 KES
3 of 3 ★ 1,175.35 KES 196.73 LOSS 1,372.08 KES — carried to next burst
Deficit carries forward

If all rounds in a burst are lost, the deficit does NOT reset. It carries into the next burst. The next trigger will start with 1,372 KES to recover and the same 50%/3-round logic applies from that new total. This is why a Stop on Loss guard is critical.

P1 and P2 — Independent Settings

Every recovery setting below exists separately for Panel 1 and Panel 2. They stay independent unless you set one panel's scope to Combined or enable an assist rule.

Setting Panel 1 Panel 2 What it does
Recovery on/off recovery_enabled p2_recovery_enabled OFF = flat bet every round (no scaling)
Bet amount bet_amount p2_bet_amount Minimum bet used when deficit is zero
Cashout target panel1_cashout panel2_cashout Auto-cashout multiplier set in the game
Profit margin recovery_profit_target p2_recovery_profit_target KES added on top of deficit in the bet formula
Recovery scope recovery_scope p2_recovery_scope individual / combined / percentage
Recovery % recovery_percentage p2_recovery_percentage % of deficit targeted per round (percentage scope only)
Assist % p1_assist_percentage p2_assist_percentage % of the other panel's deficit targeted by assist rules; P1 assist only runs when P1 is clean
Recommended starting point

P1 scope = Individual, recovery = ON — P1 is your main recovery engine.
P2 scope = Individual, recovery = OFF — P2 bets flat as a consistent hedge.

Only enable P2 recovery once you understand how deficits compound. Both panels recovering aggressively at the same time multiplies your exposure quickly.

Choosing the Right Scope

Scope Best when… Risk level
Individual P1 and P2 behave independently; you want each panel to only account for its own history Low
Combined P1 wins often enough to cover both panels' losses; you want one clear-all mechanism Medium
Percentage 50–70% You want gradual recovery spread over multiple rounds; bet sizes stay smaller each round Medium
Percentage 100% Full recovery on every round — same as Individual but with last-round enforcement; maximum bet sizes High

Safety Guards

Recovery bets grow as the deficit grows. Without limits, a bad run can make bets unaffordably large. Always configure at least one of these:

Stop on Loss
Global stop-loss (KES)
The bot stops completely when your cumulative P&L falls below this value. A negative number — e.g. -200 means stop after losing 200 KES total since the session started.
Fires immediately — no more rounds placed after this threshold.
Stop on Consecutive Losses
Loss streak limiter
Stops the session after N rounds in a row that all ended negative. A clean "I give up on this bad run" circuit-breaker that doesn't require calculating total KES.
Set to 0 to disable. Recommended: 4–6 for most strategies.
Stop on Profit
Take-profit guard
The bot stops when cumulative P&L reaches or exceeds this value. Lock in your gains — don't let a winning session turn into a losing one.
Fires after a round completes. Recommended: 2–3× your stop-loss value.
Burst Cooldown
Post-burst skip rounds
After each betting burst (win or exhaust), skip this many watch rounds before looking for the next trigger. Prevents re-triggering immediately after a loss.
Set to 0 to disable. Recommended: 3–5 after aggressive strategies.

Trigger Conditions

The bot only enters bet mode when a trigger fires. Two types exist — you can use one or both.

High Crash Trigger
Last crash above X×
After any round that crashes above your threshold (default 9×), the bot assumes the next sequence is more likely to stay lower, and begins betting.
trigger fires when: crash_mult > HIGH_CRASH_TRIGGER
Low Streak Trigger
N consecutive low crashes
When the last Window rounds all crashed at or below the Low Streak Max (default: 8 rounds all ≤ 3×), the bot reasons that a higher crash is overdue and starts betting.
trigger fires when: all of last N crashes ≤ LOW_STREAK_MAX
Trigger mode

Both — fires on either condition (more triggers per hour).
High crash only — only the first trigger type.
Low streak only — only the second trigger type.

Important reminder

Aviator is a provably fair random number game. No strategy can predict crash multipliers. The bot automates a disciplined, rule-based approach to bet sizing and risk management. It cannot guarantee profit. Always set a stop-loss you can afford to lose, and test with Demo mode before using real money.
